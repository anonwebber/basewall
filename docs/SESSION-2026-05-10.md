### 3. v4 Hook fee split LOCKED
**5% total swap tax**:
- 2% → dev wallet (ETH, recurring stream)
- 1% → brick owner pool (Sushi-style accumulator, distributed by **brick count, NOT activity**)
- 2% → LP auto-compound (deepens floor)

Architecture: pool LP fee tier 2% + hook surcharge 3% via afterSwap delta.

### 4. Domain decision
- **Primary: `basewall.fun`** (chain-tied, clear, scalable)
- **Alias: `xwall.fun`** (302/301 redirect — marketing angle for X banner pitch)
- User to buy BOTH

### 5. X banner angle (NEW)
- X banners are 1500×500 = 3:1 ratio
- 10×3 brick cluster = 3:1 = perfect fit
- 30 bricks × 0.001 ETH = **0.03 ETH per X banner cluster (~$90)**
- Dedicated UI flow: "upload your X banner" → auto-validates 10×3 selection
- Marketing copy: *"upload your X banner to the wall. forever onchain."*

### 6. Random allocation (NEW)
- Confirmed: random brick allocation via committed seed (no Chainlink VRF — gameable but fine for memecoin, no exploit vector)
- Dual-path mint:
  - `mint(quantity)` → random bricks, 0.001 ETH each
  - `mintBannerCluster()` → contiguous 10×3 = 30 bricks, 0.03 ETH flat (raster-scan order, skips reserved)
- Why: pure random kills X-banner UX; pure sequential kills secondary volume. Dual-path captures both.
