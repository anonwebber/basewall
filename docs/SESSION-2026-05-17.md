# basewall — Session log 2026-05-17

## The pivot — token + hook scrapped, NFT-only with rentals

After re-examining the v1.0 token economy (1B $WALL, Uniswap v4 hook with 5% swap tax, Sushi accumulator, 600M+1ETH LP, UNCX lock), we cut all of it. Two reasons:

1. **70% of the build complexity** lived in the token side (hook address mining, v4 integration, LP seeding, accumulator math, peer review). NFT-only ships in ~7 days instead of 17.
2. **The token was the most likely thing to *not ship*.** Hook bugs are existential. Without it, day-1 risk drops from "memecoin launch theater" to "NFT collection." Net of complexity: dramatically more shippable.

We replaced the recurring-revenue mechanism (2% dev fee on $WALL swaps) with a **rental marketplace**: brick owners list their brick for monthly rent, advertisers pay upfront, owner keeps 90%, treasury takes 10%. Income tied directly to wall usage, not memecoin volume.

  - **Paid phase**: 8,600 @ 0.0005 ETH (random alloc, 100/wallet cap)

### Pricing
- Mint price: **0.0005 ETH** (halved from 0.001)
- Banner cluster (10×3 = 30 bricks): **0.015 ETH** flat (halved from 0.03)
- Per-brick USD: ~$1.50 (at $3k ETH)

### Day-1 treasury
- 8,600 × 0.0005 = **4.3 ETH** (~$12.9k @ $3k ETH)
- No LP needed (no token), so 4.3 ETH net
- vs v1.0: 8.1 ETH net but with 17-day build + hook bug risk
- Treasury comeback strategy: post-mint reward drops to early minters (TBD — possibilities: free banner vouchers, retroactive corner-brick airdrops from reserve, advertiser referral bonuses)

### Free phase (the stealth launch hook)
- 1,000 bricks, 1 per wallet, free
- Gates: EOA only (`code.length == 0`), `tx.origin == msg.sender` (no router-wrap sybils), balance ≥ 0.01 ETH
- **No $WALL airdrop** for free bricks (there's no $WALL period now — but the principle stands: free = brick only)
- **No Twitter, no website URL public** until 1,000 free mints complete
- `websiteURL()` reverts with `WebsiteNotRevealed` until the 1,000th free mint
- `WebsiteRevealed(string)` event fires on mint #1000 — signals to indexers/scanners that paid phase is opening
- Owner can also `setWebsiteURL` after free-phase fills to trigger reveal manually (fallback if reveal-on-mint race didn't fire)

### Rental marketplace
- `listForRent(id, monthlyRentWei, minMonths)` — owner-only
- `rent(id, nMonths, adURI)` — advertiser pays upfront for nMonths
- 10% treasury cut (capped at 20% via `setTreasuryRentBps`)
- Pull-payment model: owners + treasury claim via `withdrawRentalEarnings()`
- `tokenURI` returns `adURI` during active rental, owner's `contentURI` otherwise
- Owner CAN'T `updateContent` mid-rental (advertiser is paying for that screen-time)
- Rental survives NFT transfer (new owner inherits both the active rental + future income)
- Anyone can call `endRental(id)` after expiry — flips tokenURI back to owner's content
- Content moderation: frontend filter only (chain stays immutable)

### Royalty
- 5% on secondary, ERC-2981, paid to treasury

## Open questions

- **Reveal mechanism timing**: should there be a hard timeout (e.g., 7-day auto-open paid phase even if free phase stuck)? Current design only auto-reveals on mint #1000 or manual owner `setWebsiteURL`. Adding a `block.timestamp >= deployTime + 7 days` fallback inside `mint()` would be a 3-line patch — defer to next session if we want it.
- **Cluster mint discovery UX**: the contract requires caller to pass `(startX, startY)`. Frontend needs an indexer-backed "find me an available 10×3 spot" helper. Trivial server-side, just noting it.
- **Wallet-quality gate edge cases**: smart-contract wallets (Safe, Coinbase Smart Wallet, etc) will fail the EOA check. For free phase, that's intentional (they can pay in the paid phase). Worth a note in the launch copy: "free phase: EOAs only."
- **Multisig as deployer**: if a Safe deploys Brick, then `owner() = Safe` and `mintCornerReserves()` must be called via Safe tx. Deploy script as written calls it in the same broadcast as deploy — works for an EOA deployer. If Safe-deployed, split into two txs.
